Private beta

Install 8mem on your machine.

This page is for invited testers with approved access to the private 8mem GitHub repo. It contains no passwords, tokens, API keys, or deploy secrets.

Start install Troubleshooting
Access required

Before you start

You need GitHub access to tempomesh/8mem. If you are not invited to the private repo, the clone command will fail.

Safe by design

No secrets on this page

Do not paste GitHub tokens, Telegram bot tokens, passwords, or API keys into any shared chat. Authenticate locally on your own machine.

Tester checklist

What you need before running this

5 minutes
  • Operating system: macOS or Linux recommended for this private beta. Windows users should use WSL2.
  • GitHub: you must have access to the private tempomesh/8mem repo.
  • Terminal tools: git, python3, and preferably gh GitHub CLI.
  • Telegram: optional. You can skip Telegram during setup and add it later.
  • Ollama/local LLM: optional. Install later if 8mem doctor says the LLM backend is missing.
1

Authenticate GitHub

If your machine is not already authenticated with GitHub, run:

gh auth login gh auth setup-git

If gh is not installed, use GitHub's official CLI installer for your operating system. Do not share tokens with anyone.

2

Clone the private repo

git clone https://github.com/tempomesh/8mem.git cd 8mem
3

Run the installer

bash ./install.sh

During setup, choose how you want to test first. If you are unsure, choose 1.

How do you want to use 8mem first? 1. Browser UI only (recommended first) 2. Telegram bot 3. Both browser UI and Telegram 4. Skip optional setup Choose [1]:
1. Browser UI onlyBest first QA path. No Telegram token needed. You can use the dashboard and local Test Chat.
2. Telegram botUse this only if you already have a BotFather token. A public HTTPS URL is needed for live webhooks. You can add it later with 8mem setup --mode telegram.
3. BothSets up local browser UI and Telegram in one pass. Use this for full channel testing.
4. Skip optional setupCreates local runtime files only. Run 8mem setup --mode telegram later to add Telegram, or 8mem setup to revisit all choices.

Ollama prompt: choose N unless Ollama is already installed. If you choose Y, press Enter to accept http://localhost:11434 and the default model, or type a real value like qwen2.5:14b.

Telegram prompt: paste a real BotFather token only when choosing option 2 or 3. Press Enter to generate the webhook secret. Press Enter to skip public URL if you do not have ngrok, Tailscale Funnel, or another HTTPS URL yet.

Expected result: the installer creates a local virtual environment, installs the 8mem command, and prints next steps. It should not ask you to paste secrets into this webpage.

4

Verify the install

export PATH="$HOME/.local/bin:$PATH" 8mem doctor

Expected result: Doctor should show core checks as passing. Telegram and Ollama may appear as optional or needing setup depending on what you chose.

If your shell still says 8mem: command not found, use ~/.8mem/venv/bin/8mem doctor.

5

Start 8mem

8mem serve --host 127.0.0.1 --port 8787

Then open http://127.0.0.1:8787 on the same machine for the dashboard.

To chat in the browser UI, open http://127.0.0.1:8787/chat. Test Chat needs Ollama running; the rest of the dashboard works without Telegram.

Keep this terminal open while testing. Press Ctrl+C in that terminal to stop the server.

6

Test the local API

API_KEY="$(grep '^EIGHTMEM_LOCAL_API_KEY=' ~/.8mem/.env | cut -d= -f2- | tr -d '"')" curl -s http://127.0.0.1:8787/readyz curl -s http://127.0.0.1:8787/v1/context \ -H "Authorization: Bearer $API_KEY"

Setup writes your local API key to `~/.8mem/.env`. Keep that file private and do not paste the key into shared chats.

Expected result: /readyz returns JSON with "ok": true. /v1/context returns Engram JSON with engram_version, identity, and system_prompt_injection.

If clone fails

Check repo access

gh auth status gh auth setup-git

The repo is private. Browser access alone may not mean your terminal Git is authenticated.

If command not found

Use the installed binary directly

export PATH="$HOME/.local/bin:$PATH" 8mem doctor # If that still fails: ~/.8mem/venv/bin/8mem doctor ~/.8mem/venv/bin/8mem serve --host 127.0.0.1 --port 8787
If API returns 401

Reload the local key

API_KEY="$(grep '^EIGHTMEM_LOCAL_API_KEY=' ~/.8mem/.env | cut -d= -f2- | tr -d '"')" test -n "$API_KEY" && echo "API key loaded"

Do not print or share the key. Use it only in your own terminal for local testing.

If Telegram is not ready

Skip it first

8mem can install and run locally without Telegram. Configure Telegram later with:

8mem setup --mode telegram
If Ollama is missing

Install local model support later

curl -fsSL https://ollama.com/install.sh | sh ollama pull qwen2.5:14b
If you need to update

Pull latest and reinstall

cd 8mem git pull bash ./install.sh 8mem doctor
Success criteria

When to mark the install as PASS

PASS
  • Install: bash ./install.sh completes without a crash.
  • Doctor: 8mem doctor clearly reports the core install state.
  • Server: 8mem serve --host 127.0.0.1 --port 8787 starts and stays running.
  • Health: curl /readyz returns "ok": true.
  • Context: curl /v1/context returns valid JSON, not 401/404/500.
